## 2. Project Overview
This project demonstrates the core fundamentals of **Figma Design and Prototyping**. Instead of just a static page, we built a functional **eCommerce Mobile App Prototype** that mirrors a professional shopping experience.

---

## 3. Figma Tools Demonstrated

### ✅ Frames (Artboards)
We used four primary frames to organize the user journey:
*   **Home Screen (PLP):** The landing page showing a hero banner, categories, and product grid.
*   **Product Detail (PDP):** A screen for viewing product specifics, pricing, and size options.
*   **Cart Overlay:** A slide-in transition for managing selected items.
*   **Success Screen:** A confirmation state after "Checkout".

### ✅ Organizing Layers
Components were grouped and layered using semantic naming, mimicking a professional Figma file:
*   `Navigation/Header`
*   `Product/CardGrid`
*   `UI/ActionButtons`

### ✅ Design System (Local Styles)
We established a cohesive visual identity using consistent tokens:
*   **Colors:** A premium palette using "Vault Purple" (#7c4dff), Primary Black, and Surface Gray.
*   **Typography:** Plus Jakarta Sans (800 for headlines, 400 for body).
*   **Interactions:** Smooth transitions between screens to simulate the "Noodle" connections in Figma.

---

## 4. Technical Implementation (The Prototype)

The "Magic" of the project lies in the **Prototype Interaction**. In Figma, you would draw wires between frames. In this project, we simulated that using **TypeScript**:

| Trigger | Action | Transition |
| :--- | :--- | :--- |
| **Tap Product Card** | Navigate to Detail Frame | Slide & Fade |
| **Tap Add to Cart** | Trigger Cart State | Scale Animation |
| **Tap Checkout** | Navigate to Success | Pop Transition |
