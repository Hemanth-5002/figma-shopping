2. Project Objective

Based on the "Introduction to Figma Basics" requirements, this project transitions from a static profile to a multi-screen **Shopping Experience**. It demonstrates how to handle complex product grids and transactional flows in a design prototype.

3. Components & Layer Structure

Home (PLP)

Features a dynamic Hero Banner, Category Filters, and a 2-column Product Listing Page.

Details (PDP)

Dedicated frame for Product Detail view including sizing, price, and CTA buttons.

Success (Confirmation)

The final state of the flow, providing user confirmation and a route back to the start.

4. Prototype Logic (Interactions)

We simulated the "Noodle" connections in Figma using asynchronous event handling:

  • Trigger: Tap on Product Card -> Action: Navigate to Component Frame 2 (Detail).
  • Trigger: Tap "Add to Cart" -> Action: Trigger State Change + Transition back to Home.
  • Trigger: Header Bag Icon -> Action: Overlay Transition (Cart Screen).
  • Trigger: Tap "Checkout" -> Action: Navigate to Success Screen.
